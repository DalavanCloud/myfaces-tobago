The segment layout is a grid with 12 columns. The number and the proportion of the columns can be set with the attributes extraSmall, small, medium, large and extraLarge. The value is a semicolon separated list. Allowed values are integers from 1 to 12, 'auto' and '*'.

This example show a segment layout with four and three columns and a segment layout with 'auto' and '*'.

The sum of the values is 12.

<tc:segmentLayout extraSmall="2;3;4;3">

The sum of the values is 8. After the third element, the sequence starts from the beginning. So the fourth element got a value of 2.

<tc:segmentLayout extraSmall="2;2;4">
 
<tc:segmentLayout extraSmall="*;auto;*">

For the different kinds of screen sizes, the segment layout has five attributes: extraLarge, large, medium, small and extraSmall. The first three should be used for desktops. The last two should be used for tablets and phones. Notice, that only the browser size matters. So a small browser window on a big screen also reacts with small and extraSmall.

In the following example, all attributes are used. If you are on a desktop you can see the different behavior by changing the size of the browser window.

<tc:segmentLayout extraLarge="1" large="2"
      medium="3" small="4" extraSmall="6">
    ...

The default layout can be overwritten by the <tc:segmentLayoutConstraint/> tag. Possible attributes are extraSmall, small,medium and large.

In the following example the medium default layout of the third input field is overwritter with '3'.

<tc:segmentLayout medium="4;8">
  <tc:in/>
  <tc:in/>
  <tc:in>
    <tc:segmentLayoutConstraint medium="3"/>
  </tc:in>
  <tc:in/>
</tc:segmentLayout>
